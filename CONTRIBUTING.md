# Contributing

Thank you for your interest in Lime.

## Code of Conduct

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md). By participating, you agree to uphold it.

## Current Contribution Scope

At the moment this repository seed is documentation-first. Useful contributions include:

1. architecture corrections;
2. fact checks against official vendor documentation;
3. extraction-boundary improvements;
4. GitHub publication hardening;
5. future scaffold proposals that preserve the MVP boundary.

## Rules

1. Keep claim discipline strict.
   - implemented means backed by code and validation
   - planned means designed but not built
   - research-informed means supported by external evidence but not implemented
2. Do not widen Lime into a platform by accident.
3. Prefer the smallest sound implementation.
4. Do not include secrets, internal credentials, or private infrastructure details.
5. Keep docs aligned with current evidence.

## Pull Requests

- One concern per PR.
- Update documentation when public claims change.
- Keep repository-health files consistent.
- Include validation notes in the PR description.

## Security Issues

For vulnerabilities, follow [SECURITY.md](SECURITY.md) instead of opening a public bug report.
